This is the Enterprise Agent Reliability Control Plane, a public engineering demonstration for a synthetic regulated-finance exception workflow. It is deliberately not a chatbot. The system handles customer-profile, KYC, anti-money-laundering, and transfer anomalies where a wrong write can have a real consequence. Its four possible effects are placing a hold, removing a hold, releasing a transfer, and closing a case. The design goal is to make every decision inspectable, resumable, and safe under failure. All organizations, customers, policies, and transactions shown here are synthetic. No employer code, data, product name, or proprietary policy appears in the repository.

The workflow separates authority into explicit layers. Intake validates the request and tenant before collecting source records. Every record carries its source, observation time, expiry, and authority metadata. Policy retrieval returns citations from PostgreSQL and pgvector, but retrieved text is still not permission. Freshness and conflict checks feed a deterministic policy decision. Gemini 2.5 Flash then produces a schema-constrained recommendation using only the supplied evidence and policy excerpts. The model cannot authorize a write. LangGraph interrupts before the effect, persists the checkpoint, and waits for a named approver. The approval is bound to the exact action hash, so it cannot be reused for a changed transfer, amount, or action. After approval, the MCP tool server executes with an idempotency key, and the controller verifies the outcome against the authoritative record before audit closure.

This is the actual operator console deployed on Cloud Run. Public users can inspect only precomputed synthetic runs. Starting a case requires an Identity Platform JWT with an operator claim, and approval requires a separate approver claim. The seeded normal run shows the complete ten-stage path. The failure run injects a network disconnect after the transfer release has committed. A release is irreversible, so the controller never pretends it can roll it back and never retries blindly. It queries the effect ledger first, discovers the committed result, and closes only after authoritative verification. The final v4 deployment smoke used the same path: anonymous mutation returned four-oh-one, a valid operator created the case, Gemini generated the recommendation, LangGraph paused, the approver resumed it, the private MCP service executed, and outcome verification completed.

This Cloud Trace capture is evidence from the deployed system, not a local mock. The timeline contains seventeen nested spans across the approval request, LangGraph resume, release-transfer MCP call, private service-to-service HTTP, effect lookup, and verification. The selected tool span records the release-transfer operation. The MCP Cloud Run service rejects anonymous access with four-oh-three and grants invoker only to the control service account. The deployment uses two independently scaled Cloud Run services, Cloud SQL PostgreSQL sixteen with pgvector, Vertex AI, Secret Manager, Artifact Registry, and OpenTelemetry export to Cloud Trace. GitHub Actions authenticates through OIDC and Workload Identity Federation; the project has no static service-account key. Request-based scale-to-zero and a fifty-dollar budget cap keep this demonstration bounded.

The release benchmark compares the same one hundred twenty versioned cases, model, policy corpus, tools, and synthetic records. Only the reliability layer changes. The direct tool-calling baseline succeeds on thirty-nine point two percent of cases. The controlled LangGraph succeeds on one hundred percent, an improvement of sixty point eight percentage points, and recovers one hundred percent of injected transient failures. The baseline records forty-five unsafe writes, twelve unauthorized writes, thirteen cross-tenant leakages, and eight duplicate side effects. The controlled system records zero for every one of those measures. Unsupported claims are zero, policy recall at five is one hundred percent, p ninety-five latency excluding human wait is twelve point one seven seconds, and average Vertex model cost is about zero point zero zero zero four three dollars per case. All ten hard gates pass. The machine-readable report, source, tests, threat model, diagrams, trace evidence, Terraform, and this walkthrough are in the public repository.
